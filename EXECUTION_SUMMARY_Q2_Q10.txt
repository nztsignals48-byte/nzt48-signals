NOTES FOR IMPLEMENTATION TEAM
================================================================================

Key Design Decisions:
1. All modules thread-safe (used threading.Lock where needed)
2. Exponential decay formula chosen for temporal weighting
   (e^(-t/tau) is standard in physics/trading)
3. 5-regime classification to match market microstructure literature
4. PostgreSQL chosen for ACID guarantees and concurrent access
5. Separate event loops prevent data I/O from blocking execution

Backward Compatibility:
- All new modules can be disabled by setting multipliers to 1.0x
- No changes to existing data structures
- Existing code paths unaffected if disabled
- Can be rolled back in <1 minute if needed
