         │ │ wal_reader.py:read_day_journal()                │ │
         │ │                                                  │ │
         │ │ Parse NDJSON events:                            │ │

         │ │ }                                               │ │
         │ │                                                  │ │
         │ │ GUARANTEE: Read-only, never writes to WAL      │ │
         │ └─────────────────────────────────────────────────┘ │
         │                                                      │
         │ ┌─ Step 4: Bayesian Win Rate ──────────────────────┐│
         │ │ bayesian.py:bayesian_win_rate()                  ││
         │ │                                                  ││
         │ │ Laplace smoothing: (wins + 1) / (total + 2)     ││
         │ │   → Shrinks small samples toward 50%            ││
         │ │   → Converges to raw WR as N grows             ││
         │ │                                                  ││
         │ │ Output: BayesianResult {                        ││
         │ │   raw_win_rate,                                ││
         │ │   bayesian_win_rate,  ← Applied by engine       ││
         │ │   trade_count,                                 ││
         │ │   total_pnl,                                   ││
         │ │ }                                               ││
         │ └─────────────────────────────────────────────────┘│
         │                                                      │
         │ ┌─ Step 5: Deflated Sharpe Ratio (DSR) ──────────┐ │
         │ │ bayesian.py:deflated_sharpe_ratio()             │ │
         │ │                                                  │ │
         │ │ Bailey & López de Prado (2014) formula:         │ │
         │ │   DSR = Φ((SR* - SR₀) / σ_SR₀)                │ │
         │ │                                                  │ │
         │ │ Output: DSRResult {                             │ │
         │ │   sharpe_ratio,                                │ │
         │ │   dsr,                ← p-value [0, 1]         │ │
         │ │   is_significant,     ← DSR > 0.95?           │ │
         │ │ }                                               │ │
         │ └─────────────────────────────────────────────────┘ │
         │                                                      │
         │ ┌─ Step 6: Kelly Accelerator ──────────────────────┐│
         │ │ kelly_accelerator.py:compute_kelly_updates()     ││
         │ │                                                  ││
         │ │ Per-ticker Kelly criterion:                     ││
         │ │   f* = p - (1-p)/b                             ││
         │ │   where p = Bayesian WR, b = avg_win/avg_loss  ││
         │ │                                                  ││
         │ │ Clamped: [KELLY_FLOOR=0.02, KELLY_CEILING=0.20]││
         │ │ EWA blend: new = α*f* + (1-α)*prior            ││
         │ │                                                  ││
         │ │ Output: Dict[ticker_id, KellyUpdate] {          ││
         │ │   new_kelly,          ← Applied to order sizing ││
         │ │   prior_kelly,                                 ││
         │ │   trade_count,                                 ││
         │ │ }                                               ││
         │ └──────────────────────────────────────────────────┘│
         │                                                      │
         │ ┌─ Step 7: Exit Calibration ───────────────────────┐│
         │ │ exit_calibration.py:calibrate_exit_multiplier()  ││
         │ │                                                  ││
         │ │ Analysis:                                       ││
         │ │   • % of trades hitting Rung 5 (most aggressive)││
         │ │   • % of trades stopping out early (tight stop) ││
         │ │   • MAE & MFE metrics                           ││
         │ │                                                  ││
         │ │ Rule: If >60% hit Rung 5, LOOSEN multiplier     ││
         │ │       If many early stops, TIGHTEN multiplier   ││
         │ │                                                  ││
         │ │ Output: ExitCalibrationResult {                 ││
         │ │   new_multiplier,     ← [1.5, 4.0]            ││
         │ │   rung5_rate,                                  ││
         │ │ }                                               ││
         │ └──────────────────────────────────────────────────┘│
         │                                                      │
         │ ┌─ Step 8: Regime Hunting ─────────────────────────┐│
         │ │ regime_hunting.py:hunt_regimes()                 ││
         │ │                                                  ││
         │ │ Classify trades by regime at entry:             ││
         │ │   • bull_quiet   (↗ trend, ↓ vol)  ← Often best ││
         │ │   • bull_volatile (↗ trend, ↑ vol)              ││
         │ │   • bear_quiet   (↘ trend, ↓ vol)               ││
         │ │   • bear_volatile (↘ trend, ↑ vol) ← Often worst ││
         │ │                                                  ││
         │ │ Output: RegimeHuntResult {                      ││
         │ │   regimes: Dict[label, RegimeStats] {           ││
         │ │     is_profitable,    ← Scale position if true  ││
         │ │     win_rate,                                  ││
         │ │     avg_pnl,                                   ││
         │ │   },                                           ││
         │ │   best_regime,                                 ││
         │ │   worst_regime,                                ││
         │ │ }                                               ││
         │ └──────────────────────────────────────────────────┘│
         │                                                      │
         │ ┌─ Step 9: Alpha Sieve ───────────────────────────┐ │
         │ │ alpha_sieve.py:sieve_universe()                 │ │
         │ │                                                  │ │
         │ │ Per-ticker Information Coefficient (IC):        │ │
         │ │   IC = correlation(signal, realized return)     │ │

         │ │ Promotion rule:  IC > 0.8  → Tier 1 (Vanguard) │ │

         │ │ Output: AlphaSieveResult {                      │ │

#[derive(Debug, Clone)]
pub struct DynamicWeights {
    pub bayesian_win_rate: f64,
    pub trade_count: u32,
    pub sharpe_ratio: f64,
    pub dsr: f64,
    pub dsr_significant: bool,
    pub chandelier_atr_mult: f64,
    pub regime_best: String,
    pub regime_worst: String,
    pub regime_scales: HashMap<String, f64>,
    pub kelly_fractions: HashMap<String, f64>,
}

#[derive(Debug, Clone, Default)]
pub struct UniverseClassification {
    pub tier1: Vec<i64>,
    pub tier2: Vec<i64>,
    pub tier3: Vec<i64>,
    pub locked: Vec<i64>,
}

impl Default for DynamicWeights {
    fn default() -> Self {
        Self {
            bayesian_win_rate: 0.5,           // Safe default (no edge)
            chandelier_atr_mult: 3.0,         // Default multiplier
            regime_best: "bull_quiet",        // Baseline regime
            regime_worst: "bear_volatile",    // Worst case
            // ... empty regime_scales, kelly_fractions
        }
    }
}

FAILURE MODES & RESILIENCE
================================================================================

Scenario 1: Ouroboros crashes mid-write
┌─────────────────────────────────────────────────────────┐
│ 1. dynamic_weights.toml partially written (incomplete)   │
│ 2. Crash before fsync() called                          │
│ 3. Kernel cache loss → file on disk is corrupt          │
├─────────────────────────────────────────────────────────┤
│ MITIGATION:                                             │
│ • Engine boots, calls load_dynamic_weights()            │
│ • toml::from_str() fails on malformed TOML             │
│ • Falls back to DynamicWeights::default()              │
│ • Engine starts with conservative defaults (WR=0.5)    │
│ • No crash, trading continues                          │
├─────────────────────────────────────────────────────────┤
│ OUTCOME: ✅ SAFE — Engine recovers gracefully           │
│ LOSS: Yesterday's analysis thrown away, reverts to 50% WR
│ PROBABILITY: Low (fsync prevents ~99% of corruption)   │
└─────────────────────────────────────────────────────────┘

Scenario 2: Crontab triggers during LSE hours (current bug)
┌─────────────────────────────────────────────────────────┐
│ 1. Supercronic triggers at 18:00 UTC (18:00 London)     │
│ 2. LSE is open (08:00-16:30 London times = continuous) │
│ 3. Ouroboros checks is_lse_open(london_time_secs)      │
│ 4. Returns True → pipeline.py returns error             │
├─────────────────────────────────────────────────────────┤
│ MITIGATION:                                             │
│ • No crash, no partial writes                          │
│ • exit(1) logged to ouroboros.log                       │
│ • Old TOML files not touched                           │
│ • Engine boots next morning with yesterday's weights    │
├─────────────────────────────────────────────────────────┤
│ OUTCOME: ⚠️  PARTIAL FAILURE — Pipeline skipped         │
│ EFFECT: Weights from 2 days ago used instead of fresh  │
│ PROBABILITY: 100% (crontab is set to LSE open time)    │
│ FIX: Change crontab to 23:50 ET (03:50 UTC)            │
└─────────────────────────────────────────────────────────┘

Scenario 3: Empty WAL (no trades today)
┌─────────────────────────────────────────────────────────┐
│ 1. wal_path exists but has 0 closed_trades              │
│ 2. journal.total_events == 0                            │
│ 3. pipeline.py:96 returns error                         │
├─────────────────────────────────────────────────────────┤
│ MITIGATION:                                             │
│ • No error, clean failure                              │
│ • Old TOML files not updated                           │
│ • Engine boots with yesterday's weights (stale)        │
├─────────────────────────────────────────────────────────┤
│ OUTCOME: ✅ GRACEFUL — No trading days are skipped      │
│ EFFECT: Yesterday's parameters used (slight drift)     │
│ PROBABILITY: ~10% on low-volume days                   │
└─────────────────────────────────────────────────────────┘

Scenario 4: Ouroboros runs but writes invalid JSON
┌─────────────────────────────────────────────────────────┐
│ 1. Ouroboros crashes during toml_writer.py              │
│ 2. TOML file is valid but incomplete (missing sections) │
│ 3. Schema validation fails in Rust loader              │
├─────────────────────────────────────────────────────────┤
│ MITIGATION:                                             │
│ • Rust deserializer rejects malformed TOML             │
│ • Falls back to Default                                │
│ • Engine boots safely                                  │
│ • parameter_history/ archive partially filled          │
├─────────────────────────────────────────────────────────┤
│ OUTCOME: ✅ SAFE — Incomplete updates reverted         │
│ EFFECT: WR, Kelly, etc. revert to defaults            │
│ PROBABILITY: <1% (TOML generation is simple)           │
└─────────────────────────────────────────────────────────┘

TIMELINE & HANDOFF
================================================================================

Day N, 23:50 ET (04:50 UTC + 1):
  Ouroboros nightly runs, produces:
  • config/dynamic_weights.toml (V2.0, today's analytics)
  • config/universe_classification.toml (V2.0)
  • parameter_history/ouroboros_YYYY-MM-DD.json (archive)

Day N+1, 06:00 ET (11:00 UTC):
  Engine boots, loads:
  • dynamic_weights.toml (yesterday's V2.0)
  • universe_classification.toml (yesterday's V2.0)

Day N+1, 08:00-16:30 GMT:
  Engine runs with yesterday's weights
  Accumulates closed trades → WAL events

Day N+1, 23:50 ET:
  Ouroboros runs again, reads Day N+1 trades
  Computes new weights based on Day N+1 performance

Day N+2, 06:00 ET:
  Engine loads Day N+1 weights
  [cycle repeats]
