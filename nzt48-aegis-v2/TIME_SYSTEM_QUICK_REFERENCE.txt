ENFORCEMENT LAYERS (3-Layer Architecture)
================================================================================

LAYER 1: COMPILE-TIME (Rust) - PREVENTS BINARY BUILD IF TIME LOGIC WRONG
├─ IS_LIVE=false constant (cannot be changed without build failure)
├─ IBKR retry loop skipped in simulation (saves 225s startup delay)
└─ Debug markers: BRIDGE SPAWN STARTING/COMPLETE

LAYER 2: RUNTIME (Python) - PAUSES AEGIS IF TIME OPERATION INVALID
├─ UTC-only time reads: datetime.now(timezone.utc)
├─ Timezone conversion assertions: assert london_dt.tzinfo == london_tz
├─ Heartbeat daemon: writes every 30s (bridge health monitoring)
├─ DataManager timeout: 10s (prevents infinite hang)
└─ Audit logging: [TIME_AUDIT] for all timezone conversions

LAYER 3: TESTING (Unit + Integration) - CATCHES BUGS BEFORE DEPLOYMENT
├─ Rust tests: 50+ boundary condition tests
├─ Python tests: 50+ integration tests
└─ Coverage: LSE boundaries, BST transitions, session modes, timezone conversions

================================================================================
TROUBLESHOOTING QUICK ANSWERS
================================================================================

Q: Bridge not spawning?
A: Check /app/data/bridge_stderr.log for exceptions. Likely IBKR or DataManager.

Q: Heartbeat file not created?
A: Bridge crashed before daemon started. Check bridge stdout/stderr.

Q: Watchdog says bridge is dead?
A: Heartbeat file is stale (>90s old). Check if bridge is frozen/deadlocked.

Q: Session mode wrong?
A: System time wrong. Run `date -u` (should be UTC) and `TZ=Europe/London date`.

Q: Tests fail with timezone mismatch?
A: Found naive datetime (no timezone). Replace datetime.now() with
   datetime.now(timezone.utc).

================================================================================
ENFORCEMENT RULES (NON-NEGOTIABLE)
================================================================================

1. IS_LIVE IS ALWAYS FALSE - Compile will fail if set to true
2. ALL TIMES ARE UTC - No naive datetimes, no local time in state
3. ALL CONVERSIONS ARE LOGGED - [TIME_AUDIT] marker for every conversion
4. ALL SESSION CHANGES ARE TESTED - New logic requires unit tests
5. ALL DST TRANSITIONS HARDCODED - No dynamic calculation (2025-2032)
6. HEARTBEAT MONITORS BRIDGE - Only source of truth for subprocess health
7. TESTS PASS BEFORE DEPLOYMENT - All tests must pass (no exceptions)

TIME SYSTEM ARCHITECTURE (Visual)

   ├─→ [Assert] Trade time is in UTC
   └─→ [Log] [TRADE] timestamp + session mode
   ↓
[Monitoring] Heartbeat freshness + Time skew detection
   └─→ [Alert] Telegram notification on issue
