| 1 | G2-P1 | `tokio::sync::Mutex` across `.await` in SubscriptionManager blocks all concurrent reads; converts to serial bottleneck at high tick rate | **ACCEPTED-P0** | A Mutex held across an await point prevents any other Tokio task from acquiring it until the awaited future resolves. At 100 subscriptions + US open tick burst, this stalls the entire routing engine. | 3 | G2-P3 | Cornish-Fisher Maillard (2012) domain violation: `K > S²-1` check missing — expansion is undefined and diverges below this threshold | **ACCEPTED-P1** | When excess kurtosis K ≤ S²-1, the Cornish-Fisher quantile expansion produces values outside the valid domain. | 5 | G2-P5 | Docker `/dev/shm` default 64MB — Polars mmap on `/dev/shm` causes Bus error (SIGBUS) when working set exceeds this; crashes Ouroboros mid-pipeline | **ACCEPTED-P0** | Polars uses memory-mapped files in `/dev/shm` for intermediate results. Phase 8 amendment. |
| 7 | G2-P7 | Static `σ_noise=0.03` in Gaussian-Gaussian Thompson Sampler penalises 3x ETPs which have naturally higher variance — systematically under-allocates lines to the highest-alpha instruments | **ACCEPTED-P1** | A 3x ETP has daily return std ≈ 3× the underlying (e.g., 9% daily std for a 3% underlying). | 8 | G2-P8 | Polygon EST timestamps not normalised to `Europe/London` — during BST, Polygon returns timestamps in US/Eastern which can be 5-6 hours off London time, corrupting all time-based calculations in Ouroboros | **ACCEPTED-P1** | Polygon's reference data endpoints return timestamps in US/Eastern (EST/EDT). Ouroboros features that compute "time since last trade" or "trading day boundaries" using raw Polygon timestamps will be wrong by 5-6 hours, producing stale or future-dated features. | Add `normalize_to_london(ts)` utility in Ouroboros that converts Polygon timestamps to `Europe/London` using `ZoneInfo`. | 9 | G2-P9 | WAL compaction mega-runner: the nightly compaction job can compact and delete the entry WAL event for positions held >30 days, destroying cost basis and position reconstruction on restart | **ACCEPTED-P1** | A mega-runner carry position held for 31+ days has its `PositionOpened` WAL event rotated out by the 30-day compaction.
