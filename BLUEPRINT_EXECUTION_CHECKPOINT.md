## NIGHTLY UNIVERSE SCAN FRAMEWORK

### What Happens Each Night (23:50-01:00 UK)
1. **Refresh LSE Registry**
   - Latest 12 leveraged ETPs
   - Check for delistings, splits, corporate actions
   - Update decay adjustments

2. **Scan & Rank Opportunities**
   - Screen for HIGH_CONVICTION setups (top 50 patterns)
   - Score STANDARD tier (51-200 watchlist)
   - Flag WATCHLIST (201-500 candidates)

3. **Validate Liquidity**
   - Bid-ask spreads (reject if >50 bps)
   - Volume profile (reject if <£50k avg)
   - Market hours (pre-market, regular, extended)

4. **Update Watchlists**
   - Primary (top 5 high-conviction)
   - Secondary (next 15 medium-conviction)
   - Contingency (5 backup plays)

5. **Recalibrate Thresholds**
   - Per-regime Confidence cutoffs (6.5 → 6.8 if vol high)
   - Per-regime Stop losses (1% → 2% if trending)
   - Per-regime Position sizes (Kelly → Kelly-0.5% if low conviction)

6. **Review Edge Durability**
   - Sharpe ratio trend (week over week)
   - Win rate per regime
   - Signal quality degradation
   - DSR confidence (is edge real or lucky?)

7. **Prepare Next Day**
   - Queue trades for approval
   - Pre-calculate expected entry/exit
   - Set alerts for market open

### Integration with Execution
- Morning execution (08:15 UK) uses **overnight scan results**
- Watchlist is **live-ranked** (top 3 candidates have execution priority)
- Thresholds are **regime-aware** (TRENDING_UP different from RISK_OFF)
- Edge durability **blocks trades** if edge is decaying (DSR <0.8)

## RALPH WIGGUM SAFEGUARDS (EMBEDDED)

Ralph Wiggum prompts are **wired into the execution flow**, not isolated:

### 1. Anti-FOMO Check
**When**: Confidence scorer outputs >8.5 in TRENDING_UP regime
**Prompt**: "I'm in danger" (Are you chasing a move that's already run 10%+ today?)
**Action**: Require manual override OR hold for 2 bars before entry

### 2. Revenge Trading Check
**When**: Previous trade was stopped out (loss) within last 5 minutes
**Prompt**: "Everything's coming up Milhouse" (Are you trying to recover immediately?)
**Action**: Require 10-minute cooldown + fresh signal before next entry

### 3. Averaging Down Prevention
**When**: System attempts to add to losing position
**Prompt**: "My cat's breath smells like cat food" (Are you doing something senseless?)
**Action**: Reject averaging down automatically (unless DSR >1.8)

### 4. Narrative Fallacy Check
**When**: Confidence scorer relies on single indicator (weight >50%)
**Prompt**: "I bent my Wookiee" (Are you overconfident based on one reason?)
**Action**: Reduce position size 30% until consensus is broader

### 5. Escalation Check (Operator Only)
**When**: Telegram alerts 5+ consecutive losses
**Prompt**: "In this house we obey the laws of thermodynamics!" (Stop doing things that can't work)
**Action**: Halt trading, require post-mortem analysis before resuming

## TELEGRAM ARCHITECTURE (FULLY WIRED)

### Signal Delivery Pipeline
```
Signal Generated (Phase 7, confidence score)
    ↓
Qualification Gate (Phase 8, pre-conditions check)
    ↓

    ├─ Suggested entry/exit

    ↓
Grafana Dashboard (Real-time monitoring)
    ├─ Signals/hour
    ├─ Delivery success rate
    ├─ Avg latency
    └─ Failed signal alerts
```
